I know how to work with the following languages, tools & services:

Persist your data

In order for any changes in the website to be made, new data has to be written somewhere. This is achieved with the use of external databases, that connect to the front-end and let these changes be displayed in an accurate and elegant manner.

I use a No-SQL database called mongoDB to achieve this, along with a related program called Studio3T, a GUI that allows to view and modify database data, without having to use the command line.

Secure your users

One of the best options for use authentication is OAuth, a service that allows users to authenticate through their preferred platform (i.e. Google, facebook, etc.). Quick and trustful.

For creating and managing users sessiones I use Passport, which lets the web app create dynamic tokens (also known as cookies) during the time the user is active, and make it behave differently depending on wether the user is logged in or not, or is a specific user.
